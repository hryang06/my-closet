# My Closet — URL 기반 사이즈 추천 spec

## 개요

의류 제품 URL을 입력하면 제품 정보와 사이즈 테이블을 추출하고, 저장된 신체 사이즈를 기반으로 추천 사이즈와 이유를 제공한다.

## 범위

### 포함
- URL 입력 → 제품명·브랜드·제품 설명·사이즈 테이블 파싱 및 표시
- 신체 사이즈 입력 (키·몸무게·가슴 둘레·허리 둘레·신발 사이즈 — 하나 이상 입력 시 저장 가능) 및 브라우저 로컬 저장
- 사이즈 추천 결과 — 추천 사이즈 + 이유 텍스트
- URL 지우기(리셋) 기능

### 제외
- 프롬프트 프리셋 (요약/번역/설명) — 사이즈 추천 컨셉과 무관, idea 단계에서 제거 결정
- 옷장 저장·관리 — MVP 검증 후 확장
- 로그인·계정 — 개인 도구, 로컬 저장으로 충분
- 브라우저 익스텐션 — 학습 복잡도 불필요하게 높임

---

## 시나리오

### 1. URL 입력 → 제품 정보 + 사이즈 테이블 파싱 성공

- **Given** — URL 입력창이 비어 있는 초기 화면
- **When** — 사이즈 테이블이 있는 의류 제품 URL 입력 후 조회
- **Then** — 제품명·브랜드가 헤더에 표시되고, 사이즈 테이블이 렌더링된다. "사이즈 추천" 버튼이 화면에 나타난다.

성공 기준:
- [ ] 조회 시작 → 로딩 인디케이터가 표시된다
- [ ] 파싱 완료 → 제품명이 화면에 표시된다
- [ ] 파싱 완료 → 브랜드명이 화면에 표시된다
- [ ] 파싱 완료 → 제품 설명이 제품명과 사이즈 테이블 사이에 표시된다
- [ ] 파싱 완료 → 사이즈 라벨과 실측치가 담긴 테이블이 표시된다
- [ ] 사이즈 테이블 표시 후 → "사이즈 추천" 버튼이 화면에 보인다

---

### 2. 사이즈 테이블이 없는 제품 URL

- **Given** — URL 입력창에 사이즈 정보가 없는 의류 제품 URL
- **When** — 조회
- **Then** — 제품명·브랜드는 표시되고, 사이즈 섹션에 찾을 수 없다는 안내가 표시된다. "사이즈 추천" 버튼은 비활성화된다.

성공 기준:
- [ ] 파싱 완료 → 제품명·브랜드가 화면에 표시된다
- [ ] 파싱 완료 → 제품 설명이 제품명과 사이즈 섹션 사이에 표시된다
- [ ] 파싱 완료 → 사이즈 테이블 대신 "사이즈 정보를 찾을 수 없습니다" 안내 문구가 표시된다
- [ ] 사이즈 정보 없음 → "사이즈 추천" 버튼이 비활성화 상태로 표시된다

---

### 3. URL fetch 실패

- **Given** — URL 입력창에 잘못된 URL 또는 봇 차단 쇼핑몰 URL
- **When** — 조회
- **Then** — 에러 메시지가 표시되고, 제품 정보 영역은 비어 있다.

성공 기준:
- [ ] 잘못된 형식의 URL → 에러 메시지가 표시된다
- [ ] 봇 차단 등 fetch 실패 → 에러 메시지가 표시된다
- [ ] 에러 상태 → 제품명·브랜드·사이즈 테이블이 표시되지 않는다

---

### 4. 사이즈 추천 버튼 클릭 — 신체 사이즈 미저장

- **Given** — 제품 정보와 사이즈 테이블이 표시된 상태, 저장된 신체 사이즈 없음
- **When** — "사이즈 추천" 버튼 클릭
- **Then** — 신체 사이즈 입력창이 표시된다

성공 기준:
- [ ] 신체 사이즈 미저장 → 추천 버튼 클릭 시 신체 사이즈 입력창이 나타난다
- [ ] 입력창에 키·몸무게·가슴 둘레·허리 둘레·신발 사이즈 필드(모두 선택)가 보인다
- [ ] 모든 필드가 빈 상태 → 저장 버튼이 비활성화된다

---

### 5. 신체 사이즈 저장

- **Given** — 신체 사이즈 입력창이 표시된 상태
- **When** — 키·몸무게 입력 후 저장
- **Then** — 입력창이 닫힌다. 페이지 새로고침 후에도 저장된 값이 유지된다.

성공 기준:
- [ ] 키·몸무게 입력 후 저장 → 입력창이 닫힌다
- [ ] 페이지 새로고침 후 신체 사이즈 입력창 재오픈 → 저장된 키·몸무게 값이 채워져 있다
- [ ] 가슴·허리 선택 입력 후 저장 → 새로고침 후 해당 값도 채워져 있다

---

### 6. 사이즈 추천 실행 — 신체 사이즈 저장된 경우

- **Given** — 제품 정보와 사이즈 테이블이 표시된 상태, 신체 사이즈 저장됨
- **When** — "사이즈 추천" 버튼 클릭
- **Then** — 추천 사이즈와 이유 텍스트가 표시된다

성공 기준:
- [ ] 추천 버튼 클릭 → 로딩 인디케이터가 표시된다
- [ ] 추천 완료 → 추천 사이즈 라벨(예: "M")이 화면에 표시된다
- [ ] 추천 완료 → 추천 이유 텍스트가 함께 표시된다 (제품 설명·사이즈 테이블·신체 사이즈를 종합한 근거 포함)

---

### 7. URL 리셋

- **Given** — 제품 정보가 표시된 상태
- **When** — 지우기 버튼 클릭
- **Then** — URL 입력창과 제품 정보 영역이 초기 상태로 돌아간다

성공 기준:
- [ ] 지우기 버튼 클릭 → URL 입력창이 비워진다
- [ ] 지우기 버튼 클릭 → 제품명·브랜드·사이즈 테이블이 사라진다
- [ ] 지우기 버튼 클릭 → 사이즈 추천 결과가 표시된 경우에도 함께 사라진다
